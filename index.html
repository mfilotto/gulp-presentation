Build system gulp

De l'utilité d'un build system


  1. Un ensemble de tâches (task runners)
  2. automatise des opérations répétitives
  5. fonctionne en tandem avec un gestionnaire de paquets


Les concepts de gulp


  1. Basé sur du code plutôt que de la configuration
  2. Programmation impérative plutôt que déclarative
  3. Repose sur node et les paquets npm
  4. Représenté comme une fabrique / chaîne de montage
  5. Traitements des fichiers à la chaîne

L'ADN de gulp


  1. npm pour l'installation des dépendances
  2. vinyl et adapter vinyl-fs pour l'accès aux fichiers
  3. orchestrator pour l'orchestration des tâches
  4. une CLI pour le lancement des tâches

Les éléments constitutifs de gulp


  1. le fichier package.json
  2. le fichier gulpfile.js
  3. contient des tâches (task)
  4. qui utilisent le streaming system

L'installation des dépendances avec npm


  1. npm init
  2. npm i -D maDependance

Anatomie d'un fichier gulpfile


La dépendance entre les tâches


  1. Attention liste non ordonnée, exécution en parallèle possible donc non prédictible

  2. gulp.task('hello', function () { console.log("Hello, "); }); gulp.task('world', function () { console.log("world!"); }); gulp.task('default', ['hello', 'world']);

La streaming system


  1. API basée sur le pattern method chaining
  2. lecture de fichiers en entrée avec les glob patterns : gulp.src
  3. succession de traitements sur le flux avec des plugins : gulp.pipe
  4. production de fichier en sortie : gulp.dest
  5. traintement des erreurs : gulp.on('error', fn)

Les plugins


Unité de traitement minimaliste destinée à répondre à un besoin précis : ex: minification d'un fichier
